Experiments

Conway's Game of Life

I decided to make an implementation of this famous cellular automaton discovered by the mathematician John Horton Conway. A friend of mine was talking about this and afterwards I realized how fun it would be to code it up using the canvas tag. For anyone that is new to the game of life, here is how it works: The game's universe consists of a grid containing cells. Each cell can either be alive or dead. The game also has a unit of time called a generation. The arrangement of alive and dead cells determine how the next generation will turn out. If a dead cell is surrounded by 3 alive ones, the dead cell becomes alive in the next generation. On the other hand, if an alive cell is not surrounded by 2 or 3 other alive cells, that alive cell will die off in the next generation. Every cell's fate is determined for the next generation and then painted using these simple rules to create a very interesting effect. To read more about the game of life check wikipedia. My implementation uses color to represent how old a cell is.

Julia Set

This is the Julia Set named after the French mathematician Gaston Julia. It might not be as popular as the Mandelbrot Set, but it is definitely as beautiful. The Julia Set is actually related to the Mandelbrot Set in that their generating formulae are very similar. This fractal is probably my favorite. You can find more information about the Julia Set at wikipedia. My implementation lets you explore any Julia set you want and zoom around in it. There is also a preview window that shows you what the Julia set looks like under your mouse (I particularly love this feature because it looks awesome). You can also save images you generate.

Mandelbrot Set

The Mandelbrot Set was the first fractal I learned how to make. Whenever I want to do a graphics program for fun, I usually try to create the Mandelbrot Set first. This famous fractal was named after the mathematician Benoît Mandelbrot, who studied and popularized it. This fractal is a collection of points that follow a very simple set of rules. The black area holds points that are in the "Mandelbrot Set". Aren't fractals amazing and beautiful? Learn more about the Mandelbrot Set on wikipedia. My implementation lets the user explore the Mandelbrot Set by moving around and zooming in it. You can change many different parameters and then save the image you generate when you find something cool.

Bézier Curves

Bézier Curves are a type of parametric curve. Many graphics programs use these to make interesting shapes. They are somewhat complicated to calculate out and draw. Each curve can have two or more "control points". These points can be dragged around to alter the curve. After I read about the process of how these curves are calculated, I was so impressed that I wanted to make my own implementation. The resulting canvas page lets you draw as many Bézier Curves with as many control points as your browser will allow. You can also alter how many calculations are spent on each curve to see interesting effects or to just make it look very smooth. As usual, I recommend looking at the wikipedia page to learn more about them. There are some very interesting animated GIFs on the wiki page that show the process of calculation.

Collatz Conjecture with Graphics

The Collatz Conjecture is an interesting problem named after Lothar Collatz, who first proposed it in 1937. The problem works like so: Pick any natural number (whole number) greater than 0. If the number you picked is even, divide it in half. If it is odd, multiply it by 3 and then add 1. Take the result as your new number and repeat the process. No matter what number you originally pick, you always seem to end up at 1 eventually. According to wikipedia: Paul Erdos said, allegedly, about the Collatz conjecture: "Mathematics is not yet ripe for such problems" and also offered $500 for its solution (the proof that you always hit 1). My implementation chooses a wide range of natural numbers to start with and logs how many times it needs to go through the process to hit 1. It uses these two numbers to paint a picture.
